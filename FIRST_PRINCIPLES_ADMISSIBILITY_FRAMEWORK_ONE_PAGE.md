# First-Principles Admissibility Framework

## One-Page Executive Version

**Purpose:**
A structural bullshit detector for claims, explanations, models, AI outputs, engineering proposals, institutional processes, plans, and theories.

It does not ask:

> "Does this sound right?"

It asks:

[
\boxed{
\text{What breaks first, and was that break declared before movement?}
}
]

---

# Core rule

[
\boxed{
\text{Possible} \neq \text{Repeatable} \neq \text{Scalable} \neq \text{Transferable} \neq \text{Admissible}
}
]

A claim has not earned movement until it declares:

* what carries it,
* where it holds,
* what it costs,
* how it transfers,
* how it fails,
* who repairs it,
* what budget preserves it.

---

# Five-layer audit

## 1. False Closure

**Question:**
Did the claim close too early?

**Catches:**
fluent answers, useful outputs, consensus, authority, narrative fit, symptom suppression.

**Blocked form:**

> "This looks complete, therefore it is complete."

---

## 2. First-Principles Admissibility

**Question:**
Can it exist, repeat, transfer, and endure?

[
F(X)={P,S,T,D}
]

Where:

* (P) = physical / logical possibility
* (S) = scale and repetition survival
* (T) = transfer mechanism
* (D) = decay / drift control

**Blocked form:**

> "It worked once, therefore the system can do it."

---

## 3. Upstream Formulation Failure

**Question:**
What did the formulation erase before it became clean?

**Catches:**
deleted carrier, hidden variable, unpaid correction, scale laundering, axiom theft, infinite loop, utility-as-proof.

**Blocked form:**

> "The math works because the hard part was removed before the math began."

---

## 4. Transfer Integrity

**Question:**
Did the receiver inherit the structure, or only the label?

[
T(M,S,R,C,N,B)
]

Where:

* (M) = carried object
* (S) = source state
* (R) = receiver state
* (C) = constraint clarity
* (N) = noise / distortion
* (B) = preservation budget

**Blocked form:**

> "The receiver has it because we sent it."

---

## 5. Constraint Budget

**Question:**
Who pays the bill, and can the receiver survive it?

**Catches:**
hidden cost, deferred cost, maintenance erased, correction unpaid, safety margin consumed, adoption burden hidden.

**Blocked form:**

> "No additional budget is required."

# Twelve master failure signatures

| #  | Signature                             | Core failure                                              |
| -- | ------------------------------------- | --------------------------------------------------------- |
| 1  | **False Closure**                     | Claim closes before load-bearing condition is tested.     |
| 2  | **Possible-Once -> System-Admissible** | One-off success becomes system claim.                     |
| 3  | **Deleted Carrier / Ghost Gradient**  | Output modeled without substrate.                         |
| 4  | **Transfer Failure**                  | Label transfers; constraint state does not.               |
| 5  | **Hidden Bill**                       | Cost is spent but not declared.                           |
| 6  | **Static Fantasy**                    | Time, drift, decay, and maintenance ignored.              |
| 7  | **Representation Reified**            | Model, metric, forecast, or checklist treated as reality. |
| 8  | **Evidence-State Inflation**          | Weak support promoted to closure.                         |
| 9  | **Boundary Smuggling**                | Bounded result presented as universal.                    |
| 10 | **Receiver Collapse**                 | Receiver capacity assumed.                                |
| 11 | **Governance Theater**                | Artifact treated as control.                              |
| 12 | **Behavior from Formal State**        | Rule, plan, training, or instruction treated as behavior. |

---

# Directive states

| State       | Meaning                                                                               |
| ----------- | ------------------------------------------------------------------------------------- |
| **ADVANCE** | Structurally allowed to continue under stated constraints.                            |
| **REPAIR**  | Missing carrier, budget, boundary, transfer path, evidence state, or correction loop. |
| **HOLD**    | Not enough information to judge.                                                      |
| **BLOCK**   | Structurally inadmissible under current framing.                                      |

# Core diagnostic phrases

1. **The label survived, but the structure died.**
2. **Topic transfer is not constraint-state transfer.**
3. **Possible once is not system-admissible.**
4. **Utility is not proof.**
5. **Precision without correction is unpaid constraint debt.**
6. **A signal is not control unless it can trigger action.**
7. **A right without remedy is only a label.**
8. **Safety margin is not free capacity.**
9. **A plan is not execution.**
10. **The receiver inherits a ghost when the carrier is missing.**
11. **A model is not the system.**
12. **Training is not competence until performance is demonstrated.**
13. **Documentation is not knowledge transfer until the receiver can reconstruct and act.**
14. **A forecast is not capacity.**
15. **Compliance is not governance unless it can detect, halt, correct, and enforce.**
